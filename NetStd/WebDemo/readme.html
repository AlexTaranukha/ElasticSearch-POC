dtSearch ASP.NET Core Sample

Overview

This sample demonstrates how to use the dtSearch Engine's .NET Standard API in an ASP.NET Core application.

Before running the sample, you will need to create an index of the documents to be searched. The index should be created with caching of text enabled.

To run this sample:

  1. Open the solution WebDemo.sln in Visual Studio 2017 version 15.5 or later.
  2. Select the "WebDemo" project in the solution.
  3. Click Project > Manage NuGet Packages...
  4. Restore or update any packages if needed.
  5. Edit the file appsettings.json and change the IndexTable to reference your index or indexes. (The IndexId for each index is just an identifier to use on search forms to keep the index path invisible to search users. On search forms, you can use this identifier to specify the index to search.)
  6. Build and run the application
To run this sample from the Command Prompt (Windows) or from a Terminal window (Linux/macOS):
  1. Navigate to the project folder, in the dtSearch examples/NetCore/WebDemo folder.
  2. Execute these commands:
    dotnet build
    dotnet run
  3. If the dotnet command is not found, download and install .NET here.
